# CandyTravel MP Phase 4：小程序后端接入与合法域名/环境方案

## 结论

当前已经拍板的前提：

- 前端迁移到 `uni-app`
- 首发只上：`首页 + 日历 + 计划编辑 + AI 文本提取`
- 已注册个人开发版小程序

在这个前提下，最关键的结论只有两条：

1. **数据库、后端服务、AI 服务不会上传到小程序开发页面里。**
2. **本地可以先开发，但进入真机预览 / 体验版 / 提审 / 上线之前，后端必须部署到可访问的正式环境。**

小程序上传/提审的是前端代码包；服务端、数据库、对象存储都需要单独部署。

3. 域名与证书路线

### 必须提供

4. 隐私政策 / 用户协议
5. 小程序名称、图标、简介、类目

## 环境分层建议

### 1. 本地开发环境

目标：
- 开发 `uni-app` 页面
- 联调 store / 页面逻辑
- 联调 mock API 或本地 API

允许状态：
- 后端可以先在本地跑
- 数据库可以先在本地
- AI 服务可以先用 mock 或测试 key

注意：
- 本地后端只适合开发阶段，不适合真机预览、提审和上线

### 2. 测试/体验环境

目标：
- 开发版 / 体验版小程序联调
- 产品和测试同学真机体验

必须满足：
- 后端已经部署到可访问环境
- 请求域名已经在小程序后台配置
- 域名走 `https`
- 若有文件上传/下载，也要配置对应域名

适合内容：
- 测试数据库
- 测试 AI 服务
- 基础日志与错误排查

### 3. 正式环境

目标：
- 提审
- 审核通过后正式发布

必须满足：
- 正式后端与数据库
- 正式 `https` 域名
- 合法域名已在小程序后台配置
- 用户隐私保护指引已完善
- 提审所需资料已准备齐

## 合法域名与网络约束

按微信官方文档，小程序网络访问至少要满足这些条件：

- 只能访问**已配置的服务器域名**
- `request / upload / download` 只能走 `https`
- `socket` 走 `wss`
- 不能用 `localhost`
- 不能直接配置公网 IP
- 域名需要 **ICP 备案**
- `AppSecret` 要保存在服务端，不要放在前端代码里

这意味着：

- 本地开发时可以临时联调本地服务
- 但只要进入手机真机体验和提审阶段，就必须换成正式可访问域名

## 推荐部署路线

### 推荐路线 A：先快跑

适用场景：
- 先尽快把个人开发版小程序跑通
- 首发能力只做 4 个页面 + AI 文本提取

建议组合：
- 前端：`uni-app`
- 后端：轻量 Node.js / Java 服务
- 数据库：托管 MySQL / PostgreSQL
- 文件：如果首发不做 OCR，可先不接对象存储
- AI：服务端代理调用模型

优点：
- 最接近现在团队的开发方式
- 后端、数据库、AI 都是可独立演进的

缺点：
- 你需要准备备案域名和 HTTPS

### 推荐路线 B：尽量少配基础设施

适用场景：
- 想减少前期服务器和域名配置成本

建议组合：
- 前端：`uni-app`
- 后端：微信云托管 / 云开发
- AI：仍建议通过服务端代理调用

优点：
- 对小程序联调更顺
- 域名配置负担可以相对降低

缺点：
- 团队需要接受微信云侧的部署和运维方式

## 首发阶段建议的后端能力最小集

只为了支持 `首页 + 日历 + 计划编辑 + AI 文本提取`，后端最小能力可以收在这里：

1. `Trip` 查询与更新
2. `TripDay / TripEvent` CRUD
3. `PackingItem` CRUD
4. AI 文本提取接口
5. 用户数据隔离

不建议首发一起带上：

- OCR / 图片识别
- 对象存储
- 地图与路线
- 分享协作
- 消息提醒

## 给当前项目的建议

如果目标是尽快把小程序版本推进到可联调状态，我建议：

1. `uni-app` 工程骨架先落
2. 共享数据层迁过去
3. 先把 4 个首发页面迁过去
4. 后端先做一个最小测试环境
5. 在小程序后台完成合法域名与隐私配置

顺序上，不要等所有后端能力都做完才开始小程序联调；但也不要幻想“本地后端 + 本地数据库”能直接进提审。

## 后端下一步该做什么

如果继续往前推，后端这条线建议按下面顺序执行：

### Step 1：先把测试环境立起来

目标：
- 让小程序真机 / 体验版可以访问一个真实 API

建议先确认：
- 这台机器先作为测试环境还是测试/正式共用
- `www.willer.tech` 是否已备案
- 是否已有 HTTPS 证书

### Step 2：落最小后端能力

首发只需要先支持 5 类接口：

1. `Trip` 查询与更新
2. `TripDay / TripEvent` CRUD
3. `PackingItem` CRUD
4. AI 文本提取接口
5. 用户隔离/身份识别

这一步先不要把 OCR、对象存储、地图、协作一起带上。

### Step 3：把域名和小程序后台配置打通

需要完成：
- API 域名走 `https`
- 小程序后台配置合法域名
- 若后面有上传/下载，也补上对应域名

### Step 4：准备提审必需材料

需要补齐：
- 隐私政策
- 用户协议
- 小程序名称 / 图标 / 简介 / 类目
- 测试账号 / 测试路径 / 提审说明

## 当前最缺的 4 个确认项
